# User Flow

## 1. 맛집 조회 흐름

1. 사용자는 메인 페이지에 접속한다.
2. 사용자는 일본 지도에서 관광지를 선택한다.
3. 선택한 관광지를 중심으로 지도가 확대된다.
4. 사용자는 해당 관광지 주변의 맛집 마커를 확인한다.
5. 사용자는 맛집 마커를 클릭한다.
6. 사용자는 맛집 상세 정보를 확인한다.

---

## 2. 리뷰 작성 흐름

1. 사용자는 로그인을 한다.
2. 사용자는 맛집 상세 페이지로 이동한다.
3. 사용자는 리뷰 작성 버튼을 클릭한다.
4. 사용자는 평점과 리뷰 내용을 입력한다.
5. 시스템은 해당 사용자가 이미 같은 맛집에 리뷰를 작성했는지 확인한다.
6. 작성 이력이 없으면 리뷰를 저장한다.
7. 작성 이력이 있으면 중복 작성 불가 메시지를 표시한다.
8. 저장된 리뷰가 맛집 상세 페이지에 표시된다.

---

## 3. 리뷰 수정/삭제 흐름

1. 사용자는 로그인을 한다.
2. 사용자는 자신이 작성한 리뷰를 확인한다.
3. 사용자는 리뷰 수정 또는 삭제 버튼을 클릭한다.
4. 시스템은 현재 로그인한 사용자와 리뷰 작성자가 같은지 확인한다.
5. 작성자가 같으면 수정 또는 삭제를 수행한다.
6. 작성자가 다르면 권한 없음 메시지를 표시한다.

---

## 4. AI 맛집 추천 흐름

1. 사용자는 AI 추천 페이지로 이동한다.
2. 사용자는 관광지, 메뉴 종류, 예산, 기타 조건을 입력한다.
3. 서버는 입력 조건에 맞는 맛집 후보를 DB에서 조회한다.
4. 서버는 조회된 맛집 후보 정보를 AI에게 전달한다.
5. AI는 후보 맛집 중 조건에 맞는 맛집을 추천한다.
6. 사용자는 추천 결과와 추천 이유를 확인한다.
